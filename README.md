Esse projeto é o resultado do meu trabalho de m3 da disciplina de programacao web, o objetivo era criar um sistema de ecommerce completo que tivesse tanto o front end quanto o back end e um banco de dados proprio, eu desenvolvi tudo usando node.js com express no servidor e mysql para guardar os dados, a ideia principal não era so mostrar produtos na tela mas sim gerenciar o fluxo real de uma venda/estoque que e a parte mais importante

para comecar eu precisei popular meu banco de dados entao criei uma logica de importacao, eu consumo uma api externa chamada fakestore api mas eu nao mostro os dados direto dela para o cliente, eu pego esses dados e salvo na minha tabela de produtos no mysql, fiz isso porque a api externa e apenas para leitura e eu precisava controlar o estoque, entao importo de la e defino um estoque inicial de 10 unidades para cada item, assim meu sistema tem controle total sobre o que pode ou nao ser vendido

na parte tecnica do codigo eu organizei tudo em modulos separando o que e rota do que e controller, usei a biblioteca mysql2 com promises para poder usar o async e await, isso é fundamental porque as operacoes no banco demoram um pouco e o javascript precisa esperar a resposta antes de continuar, senao ia dar erro ou vir dados vazios, alem disso eu tambem me preocupei com a seguranca dos dados entao nao deixei a senha do banco exposta direto no codigo, eu usei uma biblioteca chamada dotenv para gerenciar as variaveis de ambiente, isso significa que as credenciais do banco ficam num arquivo separado chamado .env que nao vai pro github porque eu coloquei ele no gitignore, entao -----quem for rodar o projeto precisa criar esse arquivo .env na raiz e colocar as proprias senhas la para conseguir conectar---------, no front end eu usei html css e javascript puro sem frameworks pesados, o front consome minha propria api atraves do fetch, entao quando a pagina carrega ela pede a lista de produtos pro meu servidor node e desenha os cards na tela

A parte mais complexa e que eu acho que vale destacar é o checkout e a validação de estoque, para nao ficar apenas no front end criei rotas de validacao no back, então quando o usuario clica em adicionar ou tenta aumentar a quantidade o sistema vai no banco verificar se tem aquele produto disponivel naquele momento, na hora de fechar a compra eu usei um conceito chamado transacao/transaction do banco de dados, isso garante que o pedido so é salvo se o estoque for descontado com sucesso, se der erro no meio do caminho o banco desfaz tudo com o rollback, garantindo que eu nunca venda um produto que nao tenho ou que o estoque fique negativo

alem disso fiz tambem a parte de historico onde da para buscar as compras antigas pelo e mail usando um join entre as tabelas de pedidos itens e produtos, para rodar o projeto é so instalar as dependencias com npm install criar o arquivo .env com as senhas do banco de dados, configurar o banco com o script sql(database.sql) que esta na pasta e rodar o node server.js.
